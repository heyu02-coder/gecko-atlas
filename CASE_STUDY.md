# GECKO ATLAS — 完整案例研究

## 项目摘要

GECKO ATLAS 是一座可交互的壁虎与蜥蜴数字自然博物馆。访客首先控制一只 3D 昆虫进入壁虎视野，随后可以继续探索物种图鉴、身体机制、行为实验、栖息地与保护内容。

项目不是把科普文章简单放进一个有动画的网页，而是尝试回答：**能否让输入方式本身成为自然行为叙事？**

| 项目属性 | 内容 |
| --- | --- |
| 角色 | 交互设计、前端开发、内容架构、素材研究 |
| 技术 | HTML、CSS、JavaScript、Three.js、FBX、视频 seek |
| 交付 | 响应式网站、4 个实验、3 份长篇档案、许可登记、测试矩阵 |
| 数据策略 | localStorage，本地优先，不接入远程分析服务 |

## 1. 问题定义

最初的素材是一段固定机位壁虎视频和一个拆分后的 FBX 昆虫模型。核心限制十分具体：

- 视频中的背景、404、地面、相机、壁虎身体和四肢必须固定
- 不能自动播放、循环或调用 `video.play()`
- 鼠标水平移动只能通过 `currentTime` 改变壁虎头部朝向
- 昆虫必须真实跟随鼠标，翅膀和腿由独立组件驱动
- 壁虎攻击必须以昆虫当前位置为目标，不能播放无关片段代替捕食
- 同一个网站需要从本地服务器和 Edge 链接稳定打开

这意味着项目同时包含媒体控制、3D 动画、行为逻辑和交付稳定性问题。

## 2. 目标用户与使用场景

### 公众访客

希望在短时间内获得“壁虎正在看我控制的昆虫”的直觉体验，再按兴趣进入知识内容。

### 学习者

### 招聘方或协作者

需要快速看见问题拆解、技术选择、内容责任、性能意识和验证过程。

## 4. 关键设计决策

### 决策 A：把视频当作可寻址状态，而不是播放器

鼠标 X 轴位移映射为时间偏移，并反向写入目标时间，使鼠标向左时视频推进、壁虎朝左；鼠标向右时回退。排队 seek 机制保证 seek 中只保留最新目标，避免连续赋值造成竞争。

这保留了原始固定构图，也让输入与头部朝向建立直接因果关系。

### 决策 B：昆虫不是替换光标的贴图

FBX 模型被自动居中和缩放。通过几何位置与形状复核，而不是只看名称，识别出独立翅膀和腿部组件：

- 翅膀：`part_13`、`part_14`
- 腿部：`part_0`、`part_1`、`part_3`、`part_4`、`part_5`、`part_8`

翅根建立局部枢轴，左右镜像振翅；腿部随移动速度交替摆动。昆虫转向只围绕竖直轴，上下移动只产生有限俯仰，从系统上阻止空翻和连续旋转。

### 决策 C：用状态机组织捕食，而不是堆叠动画条件

```text
稳定追踪 → 警觉 → 蓄力 → 攻击
                         ├─ 命中 → 重生
                         └─ 逃脱 → 冷却
```

距离、速度、冲刺与安全区决定状态转移。壁虎嘴部与舌头是叠加在当前构图上的程序化效果，攻击方向始终使用昆虫实时位置。

### 决策 D：科普实验必须说明模型边界

脚趾吸附实验输出的是“教学模型附着余量”，不是力学测量；伪装测试衡量网页视觉搜索，不推断真实动物视觉；体温实验使用任务生成温区，不把某个温度错误泛化为所有蜥蜴的偏好。

每个结果同时记录变量和限制，避免互动的确定性制造虚假的科学精度。

### 决策 E：地图采用代码原生示意图

重点物种地图由 SVG 形状、重点岛屿和范围标记构成，不复制第三方地图影像。页面明确说明它用于科普定位，不替代精确分布数据；正文则链接到 Reptile Database、IUCN 专家组、CITES、UNESCO 和保护机构资料。

## 5. 迭代过程

### Phase 0 — 稳定化

- 统一通过本地 HTTP 服务器运行
- 增加无缓存响应头，解决 Edge 继续加载旧效果
- 建立 WebGL 失败时的光标降级方案
- 修复模型不可见、链接打不开和本地文件行为不同的问题

### Phase 1 — 运动反馈

- 昆虫平滑跟随、悬浮、转向、冲刺和体力
- 腿部步态与速度绑定
- 壁虎稳定追踪，避免背景构图受影响

### Phase 2 — 捕食状态机

- 危险距离、警觉度、蓄力、攻击、命中、逃脱和冷却
- 张嘴与舌头效果朝向实时猎物位置
- 移除“播放另一段捕食视频后昆虫消失”的错误因果

### Phase 3 — 探索系统

- 404 热点、安全区、返回入口、彩蛋和多结局
- 捕食体验从单一挑战扩展为可探索场景

### Phase 4 — 反馈与数据

- 本地互动指标、实验结果和反馈面板
- 数据查看、导出与清除
- 不接入第三方追踪

### Phase 5A–5C — 数字博物馆

- 森林系响应式网站骨架
- 7 个物种、6 个身体专题、5 类栖息地
- 34 个响应式 WebP 文件与完整许可页
- 4 个可重复行为实验

### Phase 5D — 作品集包装

- 3 份重点物种长篇档案
- 3 张代码原生示意分布地图
- README、案例研究和设备测试矩阵
- 公开部署留到 Phase 6

## 6. 技术架构

```text
Pointer input
├─ video seek queue ──────────────┐
├─ Three.js insect transform ─────┤
├─ sprint / stamina ──────────────┤
└─ predator distance calculation ─┤
                                  ▼
                         behavior state machine
                                  │
                 ┌────────────────┼───────────────┐
                 ▼                ▼               ▼
             visual feedback   exploration     local analytics
```

模块按职责拆分，避免把所有行为放入一个全局循环：视频、3D、游戏状态、探索、数据、博物馆内容和实验各自拥有独立文件。

## 7. 内容与素材责任

所有联网获取的照片只采用 Public Domain、CC0、CC BY 或 CC BY-SA。授权清单记录作者、原始文件页、许可文本和修改说明；桌面与移动端分别提供 1600 与 800 像素 WebP。

重点物种保护状态标注评估年份。分布正文区分原生范围、引入记录与资料差异，不将示意地图表述为精确边界。

## 8. 验证结果

自动化冒烟测试覆盖 Edge 与 Chrome，分别使用桌面、平板和移动端视口：

- 首页 HTTP 与关键视频属性
- 移动端全站搜索
- 第 4 个实验入口
- 3 份重点档案、地图与来源
- 关于页和许可页
- 页面 JavaScript 与控制台错误

详细记录见 `TEST_MATRIX.md`。

## 9. 成果与价值

- 把单个视频交互扩展为拥有内容深度和重复访问路径的网站
- 将 FBX 组件分析、实时动画和状态机结合为清晰的技术叙事
- 让实验结果与科普边界同时可见，提升内容可信度
- 建立可追溯的开放素材工作流
- 用 README、案例研究与测试矩阵补齐作品集交付链

## 10. 已知边界与下一步

- 视频控制和 3D 光标主要面向精确指针设备；触屏设备保留内容浏览与实验操作，但不模拟完整鼠标捕食体验
- 分布地图是示意图，不提供 GIS 缩放、坐标查询或时间序列
- 重点档案目前覆盖 3 个代表物种，其余图鉴仍为短档案
- 当前运行方式是本地服务器，尚无公开域名

Phase 6 将专门处理公开部署、HTTPS、缓存和线上性能验证。本阶段不进行部署。
